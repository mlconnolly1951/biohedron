MSP Version 3.9.3

Release Notes

March 3, 2006

BMP (Windows bitmap) Image Support

The msdraw program can now write bmp files. Just follow the name of the filename by a space and the letters bmp. For example, msdraw —i script —r output.bmp bmp

Endian Issues

The msdraw program will now write SGI and SUN Raster image files in big endian format, regardless of the platform that the msdraw program is compiled on. Conversely, the msdraw program will now write BMP image files in little endian format, regardless of the platform that the msdraw program is compiled on. The binary surface files are written in an endian format that is dependent on the platform that msroll is compiled on, so binary files written by msroll on a big endian machine must still be read by an msdraw compiled on a big endian machine, and similarly for little endian machines.

Executable Files

The software now includes not just source files, but also executable files for the following platforms: SUN Microsystems SPARC, SGI MIPS, Apple Power Mac and Wintel. The Wintel executables require the cygwin1.dll, which can be downloaded for free from http://www.dll-files.com/

Example Scripts

In the scripts subdirectory, you will find shell scripts and msdraw scripts for creating most of the images found in the manual. Scripts ending in "sel" are for selecting which atoms in the PDB file to include in the molecule. The scripts in this directory may differ in minor ways from the examples in the manual; for example, the paths to the pdb and bin directories may vary.

Compressed Files

There is now a directory structure to the files that are included. The executables are in subdirectories sun, sgi, mac, win. The documentation is in doc. The source code is in src. The pdb files are in pdb. The scripts are in scripts. The compressed files are in msp.tar.Z for UNIX (SUN and SGI), ZIP for Windows, and StuffIt for Macintosh.
